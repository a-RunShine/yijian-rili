# v1.4.0 复盘

## 背景

v1.4.0 新增日程区域「昨天/今天/明天」三段切换，发布 dmg 资产时踩了 3 个坑。

## 踩坑

### 1. dmg 挂载不能用 `-mountpoint`

**症状**：`hdiutil attach -mountpoint ./mp` 后，AppleScript 的 `tell disk "一键日历"` 报 -1728（找不到 disk），Finder 也看不到这个 volume。

**原因**：`-mountpoint` 把 dmg 挂载到非标准位置，Finder/FinderScript 不会把它当 disk image 看待，只认 /Volumes 下的挂载点。

**正确做法**：AppleScript 布局阶段不要传 `-mountpoint`，让 hdiutil 默认挂载到 `/Volumes/一键日历`。

```bash
# 错
hdiutil attach -readwrite temp.dmg -mountpoint ./mp
# 对
hdiutil attach -readwrite temp.dmg   # 默认 /Volumes/一键日历
```

### 2. `gh release upload` 中文字符文件名丢前缀

**症状**：上传 `一键日历-v1.4.0-macOS.dmg`，release 页面 asset 名变成 `-v1.4.0-macOS.dmg`（中文被吞）。

**解决**：dmg 文件名约定用纯英文：`YijianRili-v<版本>-macOS.dmg`。`releases/v<版本>.md` 的标题、commit message、release notes 可以中文，但上传到 GH 的资产文件名必须是 ASCII。

### 3. `gh release delete-asset` 解析以 `-` 开头的 asset 名

**症状**：`gh release delete-asset v1.4.0 -v1.4.0-macOS.dmg` 报 `unknown shorthand flag: 'v'`。

**解决**：用 `--` 终止 flag 解析。`delete-asset` 接收 2 个参数，加 `-y` 用 stdin `y` 跳过确认：

```bash
echo "y" | gh release delete-asset v1.4.0 -- "-v1.4.0-macOS.dmg"
```

## 流程现状

发布 v1.4.0 实际步骤：

1. 改源码
2. `swift build` + `swift test` 验证
3. 更新 `Info.plist` 版本号（`CFBundleShortVersionString` + `CFBundleVersion`）
4. `make install`（release 构建 + 打包 `.app` + codesign + 装到 /Applications）
5. `zip -X -r` 打 zip 包
6. 写 `releases/v<版本>.md`
7. `git add -f .app/Contents/MacOS/...`（`.app` 被 .gitignore 排除，要强推内部分文件）+ `git commit` + `git push`
8. `gh release create v<版本> <zip> --notes-file releases/v<版本>.md`
9. 手动制作精致 dmg（hdiutil UDRW → AppleScript 设 Finder 布局 → hdiutil convert UDZO）
10. `gh release upload v<版本> <dmg>`

## 流程缺陷

- v1.3.0 发布时 `Info.plist` 仍是 1.2.0，**版本号没同步**。v1.4.0 已修。
- Makefile 没有 dmg target，全靠手动敲 hdiutil + osascript。出错率高（坑 1 就是这么踩的）。
- 发布前没有"检查 Info.plist 版本号 = release tag"的 gate，靠人肉记忆。
